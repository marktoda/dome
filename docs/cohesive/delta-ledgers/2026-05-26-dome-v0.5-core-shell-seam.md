## Delta at a glance

**Classification:** **Mixed.** Design-layer establishes (a) a new axiom `CORE_HAS_NO_LLM_OR_MCP_DEPENDENCY` pinning the package-bundling boundary; (b) the `ConsumerSurface` runtime construct as the protocol-agnostic four-kind aggregation; (c) the four-entrypoint package shape (`@dome/sdk` core, `@dome/sdk/workflows`, `@dome/sdk/mcp`, `@dome/sdk/cli`); (d) the Vault interface losing `aiTools` and `toolParsers` and gaining `close()`; (e) the Vault lifecycle spec (open → use → drain → close). Three new gotchas document the scars Phase B closes (transitive-llm-dependency, substrate-count-drift, ai-sdk-tool-variance). One new matrix (consumer-surface) names which entrypoint each consumer shell uses. Implementation-layer changes are deferred to `implement-cohesively`: the entrypoint split itself, `vault.close()` implementation, the `projectAiSdk` / `projectMcp` / `buildConsumerSurface` functions, four new lockstep tests (bundle-deps + gotcha-coverage + matrix-coverage + consumer-surface), and the call-site updates for every consumer of `vault.aiTools` and `vault.toolParsers`.

**Files:** 6 substrate files added (1 invariant, 3 gotchas, 1 matrix, 1 new delta ledger); 5 normative rewrites (3 specs, 1 matrix, 1 catalog); 9 frontmatter-only edits landed in repair pass 2 — `page-schema.md` (spec extension for the gotcha `coverage:` field) + 8 pre-existing gotcha docs (frontmatter additions only, no body changes). Tests are listed under §"Tests proposed" — implementation deferred to `implement-cohesively` — and not in the added-files count.

**Conceptual changes:** The **four-entrypoint package shape** is the structural realization of the package-bundling boundary the four-concept core's seal was missing. **`ConsumerSurface`** names the four-kind aggregation pattern every v1+ protocol adapter consumes. **Vault sheds aiTools/toolParsers**; the projections live in entrypoint-scoped functions consumers reach explicitly. **Vault lifecycle** is now first-class spec content (open/use/drain/close).

**Named invariants:** `CORE_HAS_NO_LLM_OR_MCP_DEPENDENCY` (axiom, added). Total invariant tier: 7 axiom + 4 shipped-default + 2 opt-in = 13.

**Behavior matrices:** `consumer-surface.md` (added). `tool-invariant-enforcement.md` updated to add the new axiom's off-matrix section.

**Gotchas:** `transitive-llm-dependency.md`, `substrate-count-drift.md`, `ai-sdk-tool-variance.md` (3 added).

**Semantic linters:** `bundle-deps.test.ts` proposed as the structural enforcement for `CORE_HAS_NO_LLM_OR_MCP_DEPENDENCY`; `dome doctor --check-substrate-counts` proposed as the mitigation for `substrate-count-drift` (deferred to v0.5.1+ per the gotcha doc).
