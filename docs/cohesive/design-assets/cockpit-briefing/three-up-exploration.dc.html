Dome design spec · 2026-06-14

One product, two media.

The cockpit and the CLI show the same daily brief. They share one status vocabulary, one type system, and one rule about color — so moving between the terminal and the phone never feels like switching apps. Below: the shared language, three cockpit directions, the harmonized terminal, and the edge states where calm products win.

01

The shared status vocabulary

One meaning, two expressions. The terminal leads each line with a glyph (meaning never rides on color alone). The web mirrors it with a colored dot in the same position — the eye scans the same left edge in both media.

stateterminalwebwhen it appears
ok ✓ a check passed · done · healthy
attention ⚠ due today · pending · needs you soon
problem ✗ overdue · error · broken link
info • a neutral open item
empty ○ nothing here · off · all-clear
action → → the single next thing to do — the only pink on screen
question ? ? Dome needs an answer from you
type
Basel Grotesk for everything human. InputMono for source paths, commands & the whole terminal — the mono is the visual thread between the two surfaces.
color
Monochrome base + status colors only. Hot-pink #FF37C7 is reserved for the brand dot and the one next action — never decoration.
posture
Verdict first, then collapse the calm. Healthy & empty things shrink to one line. The same five words name items in both media: task · follow-up · question · meeting · reminder.
