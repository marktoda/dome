## Repair-cycle scope check

The substrate-alignment-reviewer findings the repair cycle closed all tighten existing delta entries rather than create new ones:

- **B1 (Blocker, `da956b9`):** Moved hook-dispatch wrap inside `bindTools` and `bindAiSdkTools`. The ledger's `code-project-mcp` / `code-project-ai-sdk` entries said the projections "share the hook-dispatch wrap with vault.tools"; pre-repair, the projections called `bindTools(vault, writer)` without the wrap. The fix puts the wrap inside `bindTools` itself so every caller (openVault, projectMcp, projectAiSdk) inherits it. New spec subsection `sdk-surface.md §"Hook dispatch is intrinsic"` codifies the seam. New regression test `tests/integration/mcp-hook-dispatch.test.ts` (109 lines) pins it. The added test is in the same spirit as `code-consumer-surface-test` (cross-surface contract pinning) and does not introduce a new delta entry — it's an enforcement extension of `code-vault-openVault-impl`'s composition.
- **I1 (High, `48659b4`):** `DomeMcpServer({ surface })`-only. The ledger's `sdk-consumer-surfaces-section` named `DomeMcpServer` as "a thin protocol adapter over it"; the pass-1 implementation took `{ surface, vault }` because `buildToolAdapters` needed Vault. The repair hoists `buildToolAdapters` into `buildConsumerSurface` so `ConsumerSurface.tools: ReadonlyArray<ToolAdapter>` carries the protocol-rendered adapters. Spec language already permitted this shape; the repair brings the code into alignment.
- **I2 (Medium, `f134b73`):** Renamed `@anthropic-ai/sdk` → `@ai-sdk/anthropic` across 9 normative docs + 1 code-comment (`src/workflows/index.ts`). The project bundles the Vercel adapter package; the docs named the upstream SDK package. Pure doc/comment correction; no new scope.
- **I3 (Low, `eb62b1d`):** Rewrote stale `tool-adapters.ts` comment that claimed `bindTools` "transitively imports ai" — incorrect after the T8 registry split moved AI-SDK construction to `src/tools/ai-sdk-binding.ts`. Pure comment correction; no new scope.
- **I4 (Low, `1019359`):** `close()` flips a `closed` flag stopping `dispatchEvents`. The ledger's `sdk-vault-lifecycle-section` said close "releases Vault-owned resources"; pre-repair, close was a literal alias for `drainHooks`. The flag makes the spec language load-bearing. New test "dispatchEvents after close() is a no-op" added to `vault-lifecycle.test.ts`. Strengthens `code-vault-close-impl`; no new delta entry.

No Deferred items leaked (mobile/desktop/voice/web surfaces, `dome doctor --check-substrate-counts`, `parseLogEntries` consolidation, post-close vault.tools.X guard, per-bucket inbox knob). The post-close `vault.tools.X` guard the spec reserves for v1+ remains deferred — the repair only no-ops the hook-dispatch side-effects, matching what the ledger §"Deferred" anticipates.

## What looked right

- **Repair commits cite the closing finding ID and stay surgically scoped.** Each of B1/I1/I2/I3/I4 names the substrate-alignment-reviewer finding it closes in the commit subject, scopes the diff to the files the finding identified, and (where applicable) lands a regression test that would fail against pre-repair code. B1's `mcp-hook-dispatch.test.ts` and I4's added test in `vault-lifecycle.test.ts` are the load-bearing pins. This is the disciplined shape — the repair pass closed the findings without expanding scope to adjacent substrate work.
- **I1's `ConsumerSurface.tools` shape change strengthened the abstraction.** Pre-repair `ConsumerSurface.tools: BoundToolSurface` duplicated `vault.tools`; post-repair `ConsumerSurface.tools: ReadonlyArray<ToolAdapter>` carries the protocol-rendered adapter array, with the protocol-agnostic surface still reachable via `vault.tools`. The repair made `ConsumerSurface` more useful for v1+ protocol shells (HTTP, gRPC, voice) — each protocol adapter renders its own `tools` shape while the four-kind aggregation pattern stays stable. The seam landed cleaner than the original plan specified.
