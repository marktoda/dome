## Executive judgment

The substrate is overwhelmingly readable: a future contributor can reproduce the v0.5 architecture from these docs alone, name every Tool, enumerate the invariants, and trace each axiom to its enforcement path. Pass-5's new axiom (`INDEX_AND_LOG_ARE_DISPATCHER_OWNED`) is well-scoped, cleanly named, and threaded through all the right surfaces — sdk-surface, hooks, vault-layout, cli, the matrix, and `LOG_IS_APPEND_ONLY`. The single biggest gap blocking implementation: pass-5 added the 6th axiom but did **not** sweep the inventory-count line in `docs/index.md`, and a separate inbox-default contradiction inside `hooks.md` survives all five passes. Both are mechanical fixes, but both are doc-level claims a contributor *will* read at decision time and act on incorrectly.

## Blocking issues

### B1. Axiom count drift in `docs/index.md` after pass-5 promotion
- **Severity:** Blocker
- **Category:** Spec drift
- **Why it matters:** Pass-5 added a 6th axiom; the index — the doc a contributor opens first — still says "Five axioms." Pass-5 ledger explicitly updates `sdk-surface.md` axiom row count but `docs/index.md` is missing from the §"Files modified" list.
- **Evidence:** `docs/index.md:20` — "Five axioms (non-disable-able), shipped defaults (opt-out), and opt-in invariants." Contradicts 12 invariant docs (6 with `tier: axiom`) and the matrix preamble.
- **Recommended fix:** Change `docs/index.md:20` to "Six axioms (non-disable-able), shipped defaults (opt-out), and opt-in invariants. Tier shown inline." Add `docs/index.md` to pass-5 §"Files modified" in the ledger.
- **Substrate artifact:** `docs/index.md` + ledger entry.

### B2. `hooks.md` contradicts three other specs on `inbox-raw` default-on
- **Severity:** Blocker
- **Category:** Spec drift
- **Why it matters:** Pass-3 ratified `inbox/raw/` as default-on. Three specs reflect this correctly (cli.md, vault-layout.md, sdk-surface.md). The hooks spec — the canonical home for the intake mechanism — still carries the pre-pass-3 claim "`dome init` does not pre-create any intake. The vault ships bare." A contributor reading hooks.md reaches for SDK templates; reading cli.md concludes the opposite.
- **Evidence:** `hooks.md:112` ("`dome init` does not pre-create any intake. The vault ships bare") vs `cli.md:27` (".dome/hooks/intake-raw.yaml — the shipped-default intake hook"), `vault-layout.md:45` ("dome init creates [...] AND inbox/raw/"), `sdk-surface.md:216` ("intake-raw shipped-default hook"). Also `sdk-surface.md:219` "Activation of opt-in features is manual" needs to carve out intake-raw.
- **Recommended fix:** Rewrite `hooks.md:99-100, :112` to mirror cli.md: intake-raw shipped-default; other templates opt-in. Strike the contradicting "ships bare" sentence. Tighten `sdk-surface.md:219` to "Activation of opt-in features *beyond `intake-raw`* is manual" to match cli.md:35.
- **Substrate artifact:** `hooks.md`, `sdk-surface.md`.
