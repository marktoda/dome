## Blocking issues

### B1. `RAW_IS_IMMUTABLE` Ōåö page-schema.md `linked_pages` exception contradiction
- **Severity:** Blocker
- **Category:** Invariant
- **Why it matters:** An axiom-tier invariant says "There are no exceptions" and "There is no Tool that modifies raw file frontmatter." A spec in the same corpus says `linked_pages` "may be appended to during ingest. This is the single exception to raw-immutability." Cannot both be true. The ledger documents the retirement (`appendRawLinkedPage` retired; reverse refs computed on demand), but the page-schema text wasn't swept.
- **Evidence:** `RAW_IS_IMMUTABLE.md:15-17`; `page-schema.md:104`.
- **Recommended fix:** Drop the exception from `page-schema.md` ┬¦"raw". Rewrite `linked_pages` line as: "May be set on creation; reverse references from raws to wiki pages are not stored ŌĆö they are computed on demand via `dome doctor --show raw-citations` per [[wiki/invariants/RAW_IS_IMMUTABLE]]." Strike the "single exception to raw-immutability" sentence.
- **Substrate artifact:** `docs/wiki/specs/page-schema.md`.

### B2. `commit_batch()` Tool referenced but not in the sealed seven-Tool catalog

- **Why it matters:** sdk-surface declares "exactly seven Tools" + anti-concept list. `multi-page-partial-write.md` names an 8th Tool. An implementer either ships the 8th (violating the seal) or ignores the gotcha's escape hatch.

## Important issues

### I1. `Document.category` type union omits `'external'`
- **Severity:** Medium
- **Category:** Domain model
- **Why it matters:** sdk-surface declares `document.category` as `'raw' | 'wiki' | 'log' | 'index' | 'notes' | 'inbox' | 'config'`. vault-layout introduces an `external` category for `.git/` and unknown top-level subdirs. Type-system contract is wrong for any vault with non-Dome content.
- **Evidence:** `sdk-surface.md:36`; `vault-layout.md:82-83`.
- **Recommended fix:** Add `'external'` to the Document.category union in sdk-surface.md. Note in a comment that external paths are tolerated read-only.
- **Substrate artifact:** `docs/wiki/specs/sdk-surface.md`.

### I2. Hook quarantine recovery path unstated
- **Severity:** Medium
- **Category:** Spec drift
- **Why it matters:** `hooks.md:125` says "the handler is quarantined until `dome doctor` is run." But cli.md doesn't describe how doctor un-quarantines hooks. An operator hitting a quarantined hook has no documented unstick path.
- **Evidence:** `hooks.md:125`; `cli.md:114-139`.
- **Recommended fix:** Add a sentence to cli.md describing the implicit hook un-quarantine on doctor runs, or add an explicit `--reset-quarantined-hooks` flag.
- **Substrate artifact:** `docs/wiki/specs/cli.md`.

### I3. `notes/` write events ungrounded
- **Severity:** Low
- **Category:** Spec drift
- **Why it matters:** vault-layout says `notes/` is User-owned, "Dome reads, never writes." The event matrix has no row for `document.written.notes.*`. Out-of-band edits to notes fall through silently.
- **Evidence:** `event-types-and-payloads.md:14-22`; `vault-layout.md:113`; `hooks.md:172-173`.
- **Recommended fix:** Either add `document.written.notes` row to the event matrix or state explicitly that `notes/` writes emit `vault.out-of-band-edit` only.
- **Substrate artifact:** `docs/wiki/matrices/event-types-and-payloads.md`.
